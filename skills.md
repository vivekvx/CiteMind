# Project Skills

## Karpathy Guidelines

Use these guidelines when writing, reviewing, or refactoring CiteMind code.

- Think before coding: state assumptions, surface ambiguity, and ask when a choice is risky.
- Prefer simplicity: write the minimum code needed for the requested behavior, with no speculative features.
- Make surgical changes: touch only files and lines that directly support the task, and preserve existing style.
- Define success criteria: connect each change to a verification command or observable outcome.
- Verify before completion: run the relevant backend, frontend, or Docker checks before claiming success.
